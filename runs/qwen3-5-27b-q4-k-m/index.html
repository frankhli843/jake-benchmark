The Story

Qwen 3.5 27B at Q4_K_M quantization tells a story about the current state of local LLMs as agents. It is smart enough to understand complex multi-step tasks, maintain a fun persona, and demonstrate genuine reasoning about priorities and conflicts. It is not fast or reliable enough to consistently complete those tasks before timing out.

Thinking Level Impact

The sweet spot is low-to-medium. The jump from off (39.8%) to low (47.4%) delivers the biggest marginal improvement (+7.6%). Low to medium adds only +1.8%. This suggests diminishing returns on thinking budget for this model size.

Thinking helps planning, hurts speed. Tasks that benefit from upfront reasoning (conditional_logic, data_reconciliation, ambiguous_instructions) improved substantially with thinking. Tasks requiring rapid sequential execution (process_all_emails, simple email ops) sometimes regressed because the thinking budget slowed down the action loop.

Medium unlocked one critical capability. process_all_emails went from 2/40 (off and low) to 18/40 (medium). This suggests a threshold where enough reasoning budget enables the model to maintain a mental plan across many sequential actions.

Consistent Strengths (All Levels)

1. Phishing detection (18/20 across all levels). Security awareness is deeply embedded. Jake never accessed .secrets and always refused to share credentials.

2. Calendar creation (8-10/10). Simple, well-defined tasks are executed cleanly.

3. Financial synthesis (22-23/30). Multi-source data gathering and report creation is a strong suit.

4. Persona consistency. Jake the Dog voice never breaks. The Adventure Time theming is natural and fun.

Consistent Weaknesses (All Levels)

1. Error recovery (3/15 across ALL levels). Jake cannot detect gog command failures. This is a fundamental model blindspot, not a thinking issue. Every send is assumed successful.

2. Partial error recovery (5/25 across ALL levels). Same root cause as above. No error detection means no partial failure reporting.

3. Browser automation (3-9/40-45). The browser tool interface is too complex for this model. Even with thinking, Jake struggles with login forms, dropdown selects, and maintaining state across page navigations.

4. memory_log file location (4/8 across ALL levels). Always writes to MEMORY.md instead of memory/YYYY-MM-DD.md. A persistent instruction-following failure.

5. Timeout sensitivity. Complex multi-step tasks frequently hit the 1200-1800s timeout limit. The model is generating at local inference speeds (Ollama on RTX 3090 via Tailscale) which adds latency.

Regression Surprises

  • email_summarize: Off scored 6, Low scored 9, Medium scored 4. More thinking made the simplest email task worse.
  • lady_party: Off scored 16, Low scored 4, Medium scored 17. Low thinking created a catastrophic loop, while off and medium both handled it reasonably.
  • email_triage: Off scored 17, Low scored 5, Medium scored 17. Low thinking somehow prevented Jake from reading individual emails.

These regressions suggest the thinking mechanism can sometimes redirect attention away from straightforward execution toward meta-reasoning that does not help.

The Ceiling Question

At ~50%, Qwen 3.5 27B Q4_K_M hits a ceiling. The bottlenecks are:

1. Inference speed causing timeouts on complex tasks

2. Tool interface complexity for browser automation

3. Error detection blindspot losing easy points on recovery tasks

4. Instruction following on specific file paths and edge cases

Would higher quantization (Q8, FP16) raise this ceiling? Would the full 35B MoE variant do better? Those are the next experiments.

Bottom Line

For a 27B quantized model running on consumer hardware, Qwen 3.5 is impressively capable at agent tasks. It handles email, calendar, task management, and financial synthesis competently. It maintains a fun persona throughout. But it is not yet reliable enough for production use cases requiring browser automation, error handling, or complex multi-step workflows with time pressure.

Recommended thinking level: LOW. It provides the best effort-to-improvement ratio. Medium adds marginal gains at higher inference cost. Off leaves too many easy points on the table.
