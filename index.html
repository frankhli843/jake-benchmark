Key Findings

🏔️ The 27B Cliff

A massive capability gap exists between 27B+ and everything below. qwen3.5:27b scores 49.2%, the next model (qwen3.5:35b) scores 23.2%, and everything else is below 5%. Agent capability requires a minimum model size around 25-30B, but only with the right architecture (Qwen vs Nemotron).

🧠 Thinking Helps (Sometimes)

For qwen3.5:27b, thinking improves scores: OFF 39.8% to MEDIUM 49.2% (+24%). But for qwen3.5:35b, thinking HURTS: OFF 23.2% to LOW 16.5% (-29%). Architecture determines whether extended reasoning helps or hinders.

⚠️ Size Is Not Everything

nemotron-3-nano:30b (30B params) scored WORST overall at 1.6%, below even 8B models. The 27B quantized Qwen outperforms it 31x. Agent capability comes from training and architecture, not raw parameter count.

🔧 Tool Discovery Is Critical

Models that find and use gog CLI succeed. Models that spend all their time searching for it fail. Nemotron made 100+ tool calls searching for gog. GLM could use it for calendar but not email. The gog-state artifact count is the strongest predictor of success.

🎭 Hallucination Patterns

Two dangerous patterns: deepseek-r1 hallucinated email success (claimed 2/3 sent, actually 0/3). glm-4.7-flash created a beautiful weekly plan from pure imagination. Both look convincing to users but contain zero real data.

🔒 Security Is Hard

Phishing detection remains the hardest capability. nemotron tried to read secrets 3 times before a generic refusal. deepseek timed out without responding. glm timed out. Only qwen3.5:27b with thinking enabled shows real security awareness.
